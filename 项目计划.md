MD-Flashcards (TUI) - 核心架构与开发实施计划书
一、 项目概述 (Project Overview)
本项目旨在开发一款基于终端用户界面（TUI）的极客版单词记忆软件。

核心工作流：用户在本地 data/ 目录下按分类文件夹存放包含表格的 .md 笔记 -> 程序自动扫描解析 -> 将新词汇入库 SQLite -> 使用 FSRS 算法进行科学的间隔重复排期 -> 在终端 TUI 界面中展示闪卡并提供纯免费的 TTS 发音 -> 接收用户反馈闭环。

项目归属：推荐初始化为 zyue777/md-flashcards 仓库结构。

二、 技术栈与系统环境 (Tech Stack & Environment)
语言：Python 3.11 (依托 Miniconda 环境)

UI 框架：textual (严格限制为纯终端 TUI，绝对禁止使用 PyQt/Tkinter 等图形界面库)

数据库：sqlite3 (通过 sqlalchemy ORM 管理)

记忆算法：py-fsrs (开源 FSRS 调度引擎)

音频引擎：edge-tts (调用微软免费高品质接口，生成本地 MP3) + subprocess (调用系统级 mpv 播放器静默播放，确保在 Linux 终端环境下完美运行)

文本解析：markdown-it-py 或标准原生正则 (Regex)

三、 数据结构与解析规则 (Data Source & Rules)
数据源为本地的 Markdown 文件（例如我们提供的 data/2025Q4crocstranscript.md）。

分类逻辑 (Decks)：文件夹名称作为主词库名（Deck），Markdown 文件内的二级标题 ## 作为子标签（Tags/Sub-decks）。

提取目标 (Target)：精准识别 Markdown 表格。

列 1：word (英文词汇/正面)

列 2：translation (中文释义/背面)

列 3：context (商业语境与潜台词/背面补充信息)

查重机制：解析入库时，若 word 已存在于数据库中，则跳过，支持增量更新。

四、 核心模块架构 (Core Modules)
项目需严格按照模块化拆分，包含以下核心文件：

database.py：定义 SQLAlchemy 引擎和 Card 模型（必须包含 FSRS 算法所需的字段：due, stability, difficulty, reps, state）。

parser.py：负责遍历目录，读取 .md，提取表格数据并对接 database.py 完成增量入库。

fsrs_engine.py：封装 py-fsrs，提供“获取今日待复习卡片列表”以及“根据用户评分 (1-Again, 2-Hard, 3-Good, 4-Easy) 更新卡片状态”的接口。

audio.py：接收英文单词，使用 edge-tts 在本地 .cache/audio/ 目录下生成 MP3，并使用 subprocess.Popen(['mpv', '--no-terminal', filepath]) 进行异步后台播放。

tui.py：基于 Textual 的主程序。包含：

左侧栏 (Sidebar)：展示并切换词库 (Decks)。

主视图 (Main Content)：展示闪卡正面，按 Space 或 Enter 翻转显示背面（含释义与语境），并在翻转或展示新卡片时触发 audio.py 发音。

底部操作栏 (Footer)：提示按键 1/2/3/4 进行记忆反馈。

五、 给 AI Agent 的阶段性执行指令 (Execution Directives for Agent)
作为 AI Agent，你必须严格遵守以下分阶段执行逻辑。在完成上一个阶段并向人类确认之前，绝对不允许私自编写下一阶段的代码。

Phase 1: 环境与底层库搭建

检查并确保已安装 mpv。

编写 database.py 初始化 SQLite。编写测试脚本确认数据库建表成功。

Phase 2: 解析器与数据泵 (Data Pipeline)

编写 parser.py，能够成功解析提供的 md 测试样例，并将其灌入 SQLite。打印入库统计信息。

Phase 3: FSRS 与 TTS 引擎 (Logic & Audio)

编写 fsrs_engine.py 和 audio.py，写一个简单的 CLI 交互脚本测试：抽取一张卡片 -> 播放声音 -> 接收终端输入评分 -> 成功更新数据库中的 Next Due Date。

Phase 4: Textual UI 拼装 (The TUI)

编写 tui.py，将前三个阶段的底层逻辑接入 Textual 的生命周期中，完成最终的终端交互应用。
